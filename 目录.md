# 📂 VpsScriptKit 项目目录结构

项目的物理安装位置通常在：`/opt/VpsScriptKit`

1. 根目录 (项目管理与配置)

- `install.sh` - 一键安装脚本 (用户入口)。
- `update.sh` - 核心更新引擎 (处理版本对比与下载覆盖)。
- `uninstall.sh` - 彻底卸载脚本。
- `version` - 版本号文件 (单一事实来源，控制整个项目的版本)。
- `banner` - ASCII Art 字符画 Logo 文件。
- `.gitattributes` - Git 打包规则 (用于在 Release 包中剔除多余文件)。

2. 自动化工作流 (`.github/workflows/`)

- `auto-tag.yml` - 自动化标签脚本 (监测 `version` 文件变动并自动打 `Tag`)。
- `create-release.yml` - 自动化发布脚本 (监测 `Tag` 推送并自动打包 `Release`)。

3. 启动器目录 (`bin/`)

- `v` - 核心包装器 (处理软链接物理路径解析，导出 BASE_DIR)。

> 注：系统中的 `/usr/local/bin/v` 软链接会指向此文件。

4. 核心逻辑目录 (`core/`)

- `main.sh` - 脚本主程序入口 (处理 UI 循环、依赖加载、参数拦截)。
- `router.sh` - 路由分发中心 (根据用户菜单选择跳转到不同模块)。

5. 函数库目录 (`lib/`)

- `ui.sh` - UI 渲染库 (菜单、边框、提示框、颜色定义)。
- `os.sh` - 系统环境识别库。
- `utils.sh` - 通用工具函数库。
- `network.sh` - 网络/IP/流量相关函数库。
- `system.sh` - CPU/内存/负载等系统函数库。

6. 业务功能模块 (`modules/`)

- `system/` - 系统相关模块
    - `status.sh` - 系统信息展示逻辑。
    - `maintain.sh` - 系统更新中心 (二级菜单界面)。
    - `entry.sh` - 系统工具模块入口。

- `test/` - 测试相关模块
    - `entry.sh` - 各类测速/性能测试脚本入口。
    - `basic/`, `advanced/`, `docker/` 等 - 其他功能模块的目录及 `entry.sh` 占位符。

## 🚀 运行逻辑流

- 启动: 用户输入 `v` → `/usr/local/bin/v` (软链接) → `bin/v` (包装器解析路径) → `core/main.sh`。
- 交互: `main.sh` 调用 `ui.sh` 画图 → 用户输入选项 → `router.sh` 根据选项 `source` 对应的模块。
- 更新: 用户在“更新中心”选择更新 → `maintain.sh` 调用 `update.sh` → `update.sh` 下载新版并 `exec v` 重启。
